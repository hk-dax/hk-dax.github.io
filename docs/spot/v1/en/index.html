    Rate limit rule

    The rate limit rule limits the frequency of api access, and currently each api is limited to 10 times within 1s.

    It is suggested to adjust visiting strategy according the HTTP Header below:

    • rateLimit-times: request times allowed per rate limit time window
    • rateLimit-interval: interval of rate limit time window
    • rateLimit-remaining: the remaining count for current rate limit time window
    • rateLimit-reset: the expiring timestamp for current rate limit time window

    Authentication

    Overview

    The API request may be tampered during internet, therefore all private API must be signed by your API Key (Secrete Key).

    Each API Key has permission property, please check the API permission, and make sure your API key has proper permission.

    A valid request consists of below parts:

    • API Path: for example https://api.hk-dax.com/api/v1/order
    • API Access Key: The 'Access Key' in your API Key
    • Signature Method: The Hash method that is used to sign, it uses HmacSHA256
    • Signature Version: The version for the signature protocol, it uses 1
    • Timestamp: Unix timestamp format in milliseconds. It is useful to prevent the request to be intercepted by third-party.
    • Parameters: Each API Method has a group of parameters, you can refer to detailed document for each of them.
    • For GET request, all the parameters must be signed.
    • For POST request, the parameters needn't be signed and should be put in request body.
    • Signature: The value after signed, it is guarantee the signature is valid and the request is not be tempered.

    Signature Method

    The signature may be different if the request text is different, therefore the request should be normalized before signing. Below signing steps take the order query as an example:

    This is a full URL to query one order:

    https://api.hk-dax.com/api/v1/order?AccessKeyId=e2xxxxxx-99xxxxxx-84xxxxxx-7xxxx&SignatureMethod=HmacSHA256&SignatureVersion=1&Timestamp=1662022581951

    1. The request Method (GET or POST), append line break "\n"

    GET\n

    2. The host with lower case, append line break "\n"

    api.hk-dax.com\n

    3. The path, append line break "\n"

    /api/v1/order\n

    4. The parameters are URL encoded, and ordered based on ASCII

    AccessKeyId=e2xxxxxx-99xxxxxx-84xxxxxx-7xxxxSignatureMethod=HmacSHA256SignatureVersion=1Timestamp=1662022581951

    Use UTF-8 encoding and URL encoded, the hex must be upper case. For example, The semicolon ':' should be encoded as '%3A', The space should be encoded as '%20'.

    Then above parameter should be ordered like below:

    AccessKeyId=e2xxxxxx-99xxxxxx-84xxxxxx-7xxxx
    SignatureMethod=HmacSHA256
    SignatureVersion=1
    Timestamp=1662022581951

    5. Use char "&" to concatenate all parameters

    AccessKeyId=e2xxxxxx-99xxxxxx-84xxxxxx-7xxxx&SignatureMethod=HmacSHA256&SignatureVersion=1&Timestamp=16620225819511

    6. Assemble the pre-signed text

    POST\n
    api.hk-dax.com\n
    /api/v1/order\n
    AccessKeyId=e2xxxxxx-99xxxxxx-84xxxxxx-7xxxx&SignatureMethod=HmacSHA256&SignatureVersion=1&Timestamp=1662022581951

    7. Use the pre-signed text and your Secret Key to generate a signature

    • Use the pre-signed text in step 6 and your API Secret Key to generate hash code by HmacSHA256 hash function.
    • Encode the hash code with base-64 to generate the signature.

    4F65x5A2bLyMWVQj3Aqp+B4w+ivaA7n5Oi2SuYtCJ9o=

    8. Put the signature into request URL

    For Rest interface:

    • Put all the parameters in the URL
    • Encode signature by URL encoding and append in the URL with parameter name "Signature".

    Finally, the request sent to API should be:
    https://api.hk-dax.com/api/v1/order?AccessKeyId=e2xxxxxx-99xxxxxx-84xxxxxx-7xxxx&SignatureMethod=HmacSHA256&SignatureVersion=1&Timestamp=1662022581951&Signature=4F65x5A2bLyMWVQj3Aqp%2BB4w%2BivaA7n5Oi2SuYtCJ9o%3D

    Security Best Practice

    • It is strongly suggested to bind your IP with your API Key to ensure that your API Key can only be used in your machine. Furthermore, your API Key will be expired after 90 days if it is not bound with any IP.
    • It is strongly suggested not to share your API Key with anybody or third-party software, otherwise your personal information and asset may be stolen. If your expose your API Key by accident, please do delete the API Key and create a new one.

    Public Data

    Instruments Info

    HTTP Request

    • GET /api/v1/public/instruments

    Request Parameters

    Name Type Required Description
    instId String N Instrument ID, e.g. SPOT_BTC/USDT, if both instId and instType is sent, instType will be ignored.
    instType String Y Instrument type, currently only SPOT is supported

    Response:

    {
  "msg": "success",
  "code": 200,
  "data": [
    {
      "instId": "SPOT_ETC/BUSD",
      "instType": "SPOT",
      "baseAsset": "ETC",
      "quoteAsset": "BUSD",
      "baseTick": "2",
      "baseMin": "0.03",
      "quoteTick": "2",
      "quoteMin": "0.5"
    }
  ],
  "ts": 1679903538458
}

    Response Content

    Name Type Required Description
    instId String Instrument ID
    instType String Instrument type
    baseAsset String Base asset, e.g. BTC in BTC/USDT
    quoteAsset String Quote asset, e.g. USDT in BTC/USDT
    baseTick String Base asset tick size, e.g. 0.0001
    baseMin String Base asset minimum order size.
    quoteTick String Quote asset tick size, e.g. 0.0001
    quoteMin String Quote asset minimum order size.
